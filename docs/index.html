What It Detects
Seven categories of semantic compiler optimization changes, detected with zero false positives.
⚡
Vectorization
Detects SIMD vectorization gained or lost. Identifies <N x T> instruction patterns and dominant vector width changes (e.g., AVX2→SSE4.2).
🔁
Loop Unrolling
Identifies when loops are unrolled or rerolled by detecting structurally-similar block groups and measuring block count ratios.
📥
Function Inlining
Tracks call-site counts per callee between IR versions and cross-references whether the callee was removed from the module.
🧹
Dead Code Elimination
Detects removed unreachable blocks and eliminated branches that clang proved can never execute.
🌿
Control Flow Changes
Reports structural CFG changes: branch elimination, branchless rewrites, if-chains collapsed to select instructions.
✍️
Signature Changes
Catches ABI-breaking return type and parameter type changes between IR versions — changes invisible in a source diff.
➕
Function Add / Remove
Tracks functions added to or removed from the module — essential for detecting outlining, cloning, and dead function elimination.
📄
Three Output Formats
Output as text (ANSI-colored), JSON (machine-readable), or HTML (styled interactive report) — pick what your pipeline needs.

Key Algorithms
LCS Instruction Diff
Longest Common Subsequence on instruction patterns (opcode + type signature), not raw text. Immune to register renaming noise. O(m·n) on block pairs — fast for typical IR (2–50 instr/block).
Jaccard Block Matching
Fuzzy block matching using Jaccard index on instruction-pattern sets. Threshold ≥ 0.30 correctly pairs semantically related blocks even when their names change (e.g., vector.body → for.vector.body).
Back-Edge Loop Detection
Loop headers detected by: (1) presence of a PHI node, (2) a predecessor appearing later in block layout order. No opt binary required — pure IR analysis.
Register Canonicalization
Numbered temporaries (%5, %12) renamed to %r0, %r1, … in definition order per function. Named registers preserved for semantic value.

Quick Start
Zero dependencies. No pip. Works on Linux, macOS, and Windows (Git Bash / WSL).
bash
# 1. Clone the repo
$ git clone https://github.com/ESHWAR-404/semantic-diff-compiler.git && cd semantic-diff-compiler

# 2. Build (checks Python, runs smoke test)
$ ./build.sh

# 3. Run on a test case
$ ./run.sh testcases/tc4_vectorization/v1.ll testcases/tc4_vectorization/v2.ll

# 4. Generate an HTML report
$ ./run.sh testcases/tc1_loop_bounds/v1.ll testcases/tc1_loop_bounds/v2.ll --format html --output report.html

# 5. Run the full interactive dashboard (v2)
$ python demo.py
